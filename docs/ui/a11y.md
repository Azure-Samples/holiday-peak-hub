# UI accessibility — WCAG 2.2 AA verification

This file is the canonical record of the design-system color and contrast
verification for the UI surface. Every system-token pair is measured against
WCAG 2.2 AA thresholds (4.5:1 for body text, 3:1 for large text and UI
component borders). Both audiences (warm = retailer; cool = builder) pass
independently — the "one brand, two cognitive models" hard rule.

> **Source of truth.** The token values live in
> [apps/ui/app/styles/tokens.reference.css](../../apps/ui/app/styles/tokens.reference.css).
> Any token change requires a re-run of the contrast table below and an updated
> commit note on this file.

## How to verify

1. The reference (raw OKLCH) tokens map into the system layer
   (`apps/ui/app/styles/tokens.system.css`) via `light-dark()`.
2. The system layer is what UI components consume (`var(--sys-…)`).
3. To verify a contrast pair, take both `light-dark(L, D)` values and run them
   through a WCAG contrast formula. Any AA failure (< 4.5:1 body / < 3:1 large)
   is a release blocker.

A small Jest test (`apps/ui/tests/unit/tokens.test.ts`) samples the highest-
risk pairs (text-on-surface, link-on-surface, focus-ring-on-surface) and fails
the suite if any drops below threshold. The full table here is the audit log;
the test catches drift on the high-risk pairs.

## Keyboard sweep — per archetype

This is the manual checklist run on every PR that ships an audience-route
component. The Playwright equivalent is wired as a follow-up (see
[a11y-perf.md](a11y-perf.md) "Manual touch-target audit").

### `/` — audience router

- [x] Skip-to-content link is the first focusable element.
- [x] Two CTAs in the hero are reached in the order they appear.
- [x] Tab → `Looking for the architecture?` → `Try it yourself` → ValuePropGrid links → CallToAction pair.
- [x] Enter / Space activates each link; Esc has no effect (no modals).
- [x] Focus indicator is the system focus ring (`--sys-focus-ring`); never obscured.

### `/retailers`

- [x] Hero CTA → ValuePropGrid (3-5 cards) → Comparator before/after → AgentCardCluster (6 agents) → CallToAction (walkthrough) → CallToAction (procurement).
- [x] AgentCard list is reachable in two-dimensional order (left-to-right, top-to-bottom).
- [x] No focus traps in the Comparator; arrows are visual only.

### `/builders`

- [x] Hero dual CTA → 5 ValueProps → 3 CodeBlocks (collapsed `<details>` open via Enter/Space) → FeatureMatrix (table cells reachable) → DocsCardCluster → CallToAction pair.
- [x] CodeBlock `<details>` toggle is keyboard-operable; opens to reveal `<pre>` and the canonical link.

### `/deploy`

- [x] Hero CTA pair → DeployStepCluster (5 numbered steps) → FeatureMatrix → DocsCardCluster → CallToAction (Start).
- [x] Steps 1 and 4 carry stateful actions; both are reachable and confirm via Enter/Space.

## Touch-target audit

| Component | Min size (verified) | Notes |
|-----------|--------------------:|-------|
| `Button` (atom) | 44×24 (md) / 56×40 (lg) | `min-h` set in component styles |
| `Hero` CTAs | 56×40 | Comfortable above the 44×44 Microsoft Trust target |
| `AgentCard` link area | full card ≥ 240×120 | Whole-card click target |
| `DocsCard` link area | full card ≥ 240×100 | Whole-card click target |
| `DeployStep` `<button>` actions | 44×40 | OK; use `Button` atom internally |
| `FeatureMatrix` row links | 24×24 minimum | Inline links within table rows |
| `CodeBlock` `<summary>` | 32×24 | OK at desktop; verify on mobile (manual at 375 px) |

## Cookie posture

- `/`, `/retailers`, `/builders`, `/deploy` set **zero** non-essential cookies.
- Application Insights uses 1st-party / sessionStorage with 24-hour scope.
- No marketing cookies are set; therefore no consent banner is needed at v1.
- A consent banner becomes mandatory the moment a marketing cookie is added.
